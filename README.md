# RussianEgeBot

Telegram-бот для подготовки к ЕГЭ по русскому языку. Охватывает все 26 заданий первой части в двух
режимах — тренировка (по одному упражнению с мгновенной обратной связью) и экзамен (задание в
формате реального ЕГЭ). База — 17 140 упражнений. Подбор заданий адаптируется под историю ответов
пользователя.

Бот: [@russsian_bot](https://t.me/russsian_bot)

---

## Возможности

- Все 26 заданий первой части ЕГЭ, режимы *Drill* и *Exam*.
- Адаптивный подбор: сначала непоказанные упражнения, затем — вероятностное ранжирование решённых
  по статистике ответов.
- Гибкая проверка ответов: регистр, `ё → е`, тире ↔ пробел, несколько допустимых форм.
- Разбор с подсветкой ответа пользователя, свёрнутыми блоками (объяснение, текст, варианты) и
  исправленными предложениями — на нативных Telegram Rich Messages.
- Статистика по категориям, серии верных ответов, ежедневные стрики.

## Как устроено

### Подбор заданий

Две фазы. Пока в категории есть непоказанные упражнения — берутся случайные из них. Когда пул
исчерпан или сжат фильтрами, включается ранжирование ранее решённых:

```
S = θ · R · T
```

| | |
|---|---|
| `θ ~ Beta(n_wrong + 1, n_correct + 1)` | Байесовская оценка вероятности ошибки: сопряжённый апостериор Бета-Бернулли с неинформативным приором `Beta(1, 1)`. Сэмплирование даёт баланс exploration/exploitation — у редко решавшихся упражнений выше дисперсия. |
| `R = 1 + ln(1 + d) / 10` | Коэффициент давности, `d` — дней с последней попытки. Интервальное повторение без явного расписания; логарифм не даёт старым заданиям задавить свежие ошибки. |
| `T = 1 + 0.3 · clamp(t / median − 1, 0, 2)` | Время решения относительно медианы по категории (медиана — устойчивость к выбросам). Долгое решение — признак субъективной сложности. |

Статистика считается по скользящему окну из 5 последних попыток: уровень ученика меняется во
времени, поэтому старые наблюдения намеренно забываются.

Модель работает на уровне отдельного упражнения (либо `group_id` / типа ответа — в зависимости от
стратегии), а не на уровне правил. Планируемый следующий шаг — агрегация апостериора по правилам:
это даст перенос статистики между однотипными упражнениями.

Стратегии выбора: по группам (`select_smart_by_group`), с уникальными ответами
(`select_smart_distinct_answer`), группы по одному ответу для заданий 9–12
(`select_smart_same_answer_groups`). Фильтры компонуемые, работают одинаково в обеих фазах.

### База заданий

Упражнения собраны собственным пайплайном из открытого банка ФИПИ, вариантов СтатГрад, заданий
реального ЕГЭ и типовых сборников:

```
Источники → парсинг HTML → нормализация (юникод, регистр, символы)
          → дедупликация (точные дубли, ё/е, дефис/пробел, по объяснению)
          → обогащение через Gemini API (structured output + чекпоинты)
          → валидация → генерация SQL
```

### Рекурсивный CTE для задания 22

Экзаменационный режим задания 22 требует набрать 5 предложений, у которых средства выразительности
не конфликтуют между собой. Задача удовлетворения ограничений решается на стороне БД: дерево
строится рекурсивным CTE через `CROSS JOIN LATERAL`, совместимость проверяется пересечением
массивов (`&&`), непоказанные предложения приоритизируются прямо в рекурсии.

### Архитектура

Послойная (presentation → application → domain → infrastructure), полностью асинхронная, со строгой
типизацией и dependency injection. Логика задания отделена от вёрстки: процессор отдаёт данные
форматтеру, тот строит семантический view-model, рендерер превращает его в Rich Markdown.
Package-by-feature — схемы, процессор и форматтер каждого задания лежат в одном пакете.

```

└── bot/                  # aiogram: хэндлеры, клавиатуры, middleware
```

## Запуск

Нужны Docker и Docker Compose.

```bash
cp .env.example .env                    # BOT_TOKEN и пароли
docker compose --profile dev up -d      # postgres + redis + бот с hot-reload
docker compose --profile migrate run --rm bot-migrate alembic upgrade head
```

Разработка:

```bash
docker compose --profile test run --rm bot-test   # тесты (Postgres поднимается автоматически)
uv run ruff check src/                            # линт
uv run pyright src/                               # типы
```
